# Source Playbook

Use this file when gathering the problem statement, editorial, and backup explanations.

## General order of operations

1. Open the exact problem URL.
2. Read the statement and constraints before reading any editorial.
3. Search for the official editorial or tutorial for that exact problem or contest.
4. Search for high-quality public backup explanations only when the official source is missing, too terse, or insufficient for the user's question.
5. Reconstruct the explanation yourself only after reading the available sources.

## Codeforces

Priority order:

1. Exact problem page on codeforces
2. Official tutorial or editorial blog for the contest
3. Contest blog comments, clarifying discussion, or accepted public walkthroughs
4. Strong public videos or blog posts that explain the same idea clearly
5. Your own reconstruction

Notes:

- Official editorials may live on a contest tutorial blog rather than the problem page itself.
- Some useful explanations are in discussion threads, but treat them as secondary.
- If you cannot find the official editorial, say that clearly and continue.

## AtCoder

Priority order:

1. Exact task page
2. Official editorial page for the contest or task
3. Good public writeups and translations
4. Your own reconstruction

Notes:

- Some editorials are bilingual or have translated variants.
- If the official editorial is in a language the user does not know, summarize it in the user's chosen language.

## LeetCode

Priority order:

1. Exact problem page
2. Official solution or editorial when publicly accessible
3. High-quality discuss posts or public blogs
4. Your own reconstruction

Notes:

- Do not pretend that gated content was available if it was not.
- If only the statement is accessible, solve from first principles and be explicit about that.

## Using gpt or gemini explanations

You do not have direct built-in access to private chats or tools outside the current environment.

- If the user pastes explanation text from gpt, gemini, or another assistant, simplify and verify that text directly.
- If the user only asks for "a gpt-like or gemini-like explanation," use the best public sources you can access plus your own reasoning.
- Never imply that you directly queried an external assistant when you did not.

## Evidence discipline

- Distinguish clearly between sourced editorial content and your own inferred explanation.
- Cite the problem page and the editorial source when summarizing sourced details.
- When extending beyond the source, label it as an inference or reconstruction.
- If you are uncertain, say what is uncertain instead of filling the gap with confident guesses.
